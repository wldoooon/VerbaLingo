# Implement Redis Dirty Set Tracking Plan

## Overview
Modify the `UsageService.increment_usage` method to track active user IDs in a Redis set. This "Dirty Set" will be used by the background janitor to perform batch updates to PostgreSQL.

## Current State Analysis
- **File**: `app/services/usage_service.py`
- **Function**: `increment_usage` (Lines 116-136)
- **Status**: Increments counters and sets TTL, but does not track which users were updated.

## Implementation Approach
We will add an `await redis.sadd("usage:dirty_users", str(user.id))` call inside the `if user:` block. This ensures that every time a logged-in user's counter is touched, they are queued for a DB sync.

# Usage Consolidation Implementation Plan

## Overview
Unify the fragmented rate-limiting systems by making the `feature_rate_limit` decorator use `UsageService` for counter management and tracking.

## Current State Analysis
- **File**: `app/core/limiter/decorators.py`
- **File**: `app/services/usage_service.py`
- **Status**: Two separate systems; one handles limits, the other handles persistence. They are not connected.

## Implementation Approach
1. Modify `app/core/limiter/decorators.py` to import and use `UsageService`.
2. Update the `feature_rate_limit` wrapper to:
    - Call `check_usage_limit` before the function runs.
    - Call `increment_usage` after the function succeeds.

# Setup APScheduler Implementation Plan

## Overview
Integrate `APScheduler` into the FastAPI application to enable periodic background tasks, specifically for usage syncing.

## Current State Analysis
- **File**: `app/main.py`
- **File**: `app/requirements.txt`
- **Status**: No scheduler currently integrated.

## Implementation Approach
1. Add `apscheduler` to `requirements.txt`.
2. Initialize `BackgroundScheduler` in `app/main.py`.
3. Start the scheduler during startup and shut it down during teardown.
4. Add a placeholder job that we will link to the sync logic in the next ticket.

## Phase 1: Dependency & Lifecycle
### Overview
Add the library and wire the startup/shutdown.

### Changes Required:
#### 1. app/requirements.txt
**Changes**: Add `apscheduler`.

#### 2. app/main.py
**Changes**: Initialize and manage scheduler lifecycle.

```python
from apscheduler.schedulers.background import BackgroundScheduler

# ... inside lifespan ...
    # 3. Initialize Background Scheduler
    scheduler = BackgroundScheduler()
    # Placeholder for the sync job
    # scheduler.add_job(sync_function, 'interval', minutes=5)
    scheduler.start()
    app.state.scheduler = scheduler
    logger.info("Background scheduler started")
    
    yield
    
    # Shutdown
    if hasattr(app.state, 'scheduler'):
        app.state.scheduler.shutdown()
        logger.info("Background scheduler shut down")
```

# Research: Setup APScheduler in FastAPI

**Date**: 2026-02-06

## 1. Executive Summary
This research documents the current `lifespan` implementation in `app/main.py` and outlines the steps to integrate `APScheduler` for periodic background tasks.

## 2. Technical Context
- **Entry Point**: `app/main.py`
- **Current Lifespan**: Handles database initialization and Manticore client setup.
- **Missing Dependency**: `apscheduler` is not present in `app/requirements.txt`.

## 3. Findings & Analysis
- **Scheduler Choice**: `BackgroundScheduler` is appropriate for in-process tasks in FastAPI. It runs on a separate thread, so it won't block the main event loop.
- **Lifecycle Integration**:
    - **Startup**: `scheduler.start()` should be called after DB/Manticore setup to ensure all services are ready.
    - **Shutdown**: `scheduler.shutdown()` must be called in the `yield` cleanup block to prevent dangling threads.
- **Job Configuration**: The job will need a reference to the sync service/function, which may require manual dependency resolution since the scheduler runs outside the request context.

## 4. Technical Constraints
- **Library Version**: `APScheduler==3.10.1` or later.
- **Thread Safety**: The sync function must be thread-safe when accessing the database (using a new session for each run).

## 5. Architecture Documentation
The application uses a standard FastAPI lifespan context manager. This is the idiomatic place for background worker initialization.

# Implement Batch Sync Logic Plan

## Overview
Implement the background "Janitor" task that pulls dirty user IDs from Redis and synchronizes their usage counters to the PostgreSQL database in batches.

## Current State Analysis
- **File**: `app/services/usage_service.py`
- **File**: `app/main.py`
- **Status**: Redis is tracking dirty users, and the scheduler is running, but no sync logic exists.

## Implementation Approach
1. Swap `BackgroundScheduler` for `AsyncIOScheduler` in `app/main.py` to support async sync functions.
2. Implement `sync_all_dirty_users` in `app/services/usage_service.py`.
3. Add the job to the scheduler in `app/main.py` to run every 5 minutes.
